# Proyecto Final - Portafolio Personal

## Propósito

Este proyecto consiste en la creación y publicación de una página web básica desarrollada únicamente con HTML, como práctica del uso de Git, GitHub y GitHub Pages.

El sitio funciona como un mini portafolio personal donde presento algunos proyectos tecnológicos relacionados con Machine Learning y desarrollo en Python, así como mi participación en el hackatón de ciberseguridad HackMex y experiencias en eventos tecnológicos realizados por Google y Cisco.

El objetivo principal fue reforzar el uso de etiquetas semánticas de HTML, organizar correctamente el contenido y aplicar buenas prácticas de control de versiones con Git.

---

## Características del sitio

- Estructura HTML semántica (`header`, `main`, `section`, `article`, `footer`)
- Uso correcto de jerarquía de títulos (`h1`, `h2`, `h3`)
- Enlaces funcionales externos
- Imágenes con atributo `alt` para accesibilidad
- Formulario básico de contacto
- Favicon personalizado
- Publicación mediante GitHub Pages

## Pasos de despliegue

1. Crear un repositorio público en GitHub.
2. Desarrollar el archivo `index.html` en la raíz del proyecto.
3. Realizar commits descriptivos utilizando Git.
4. Subir los cambios al repositorio remoto con `git push`.
5. Activar GitHub Pages desde:
   - Settings → Pages  
   - Seleccionar la rama `main`  
   - Carpeta `/root`
6. Verificar que la URL pública funcione correctamente en modo incógnito.

## Aprendizajes

Durante el desarrollo del proyecto reforcé:

- La estructura básica de un documento HTML.
- La importancia de la jerarquía semántica.
- El uso de Git para organizar cambios mediante commits claros.
- El proceso de despliegue de un sitio estático con GitHub Pages.
